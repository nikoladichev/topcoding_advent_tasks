# Application User Management Admin Panel
## Requirements: 
### Functional
The system must have an API | User Interface to manipulate user roles - it should be able to provide CRUD operation on the user's roles. Users should have a name, email and a bio.
### Non Functional
The system supports up to 50000 users and 50000 roles.
The system should be able to return a result withing 50ms

## Model Design
The DB can be relational or a key-value type (like DynamoDB). If all users do a read request at once a single db instance should be able to handle about 50000 rows of data per second and the key-value db should take about twice as much. At high loads this may be insufficient to get below the 50ms requirement.
